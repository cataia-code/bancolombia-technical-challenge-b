------------------

- La documentacion ya no presenta metricas porcentuales como argumento comercial.
- Los umbrales siguen configurados en el pipeline para proteger regresiones.
- Python local verificado: `C:\Users\ASUS\AppData\Local\Programs\Python\Python313\python.exe` responde Python 3.13.0; tambien existen Python 3.11.9 y 3.10.11.

Docker Compose
--------------
- `docker compose -f poc\docker-compose.yml config` valida correctamente la configuracion.
- n8n queda pinneado en `n8nio/n8n:2.30.5` para evitar drift por `latest`.
- `docker compose -f poc\docker-compose.yml up -d --build` construye y levanta:
  - `taskbot-advisor` en `http://localhost:8000`
  - `taskbot-n8n` en `http://localhost:5678`
- `http://localhost:8000/health` responde `{"status":"ok"}`.
- `http://localhost:5678/healthz` responde `{"status":"ok"}`.

n8n local
---------
- El workflow local se mantiene en `poc/n8n/workflow.json`.
- El flujo depende solo del workflow local montado en Docker.
- El nodo "Extraer hallazgos" expone `run_id`, `headlines`, `resumen` y el desglose `ola_3`.
- Validacion CLI en Docker: importar `workflow.json` y ejecutar por ID termina con `status: success`.
- CI agrega smoke Docker: levanta `advisor` + `n8n`, valida health de ambos servicios e importa/ejecuta
  el workflow por ID con `n8n import:workflow` y `n8n execute`.
- Smoke local ejecutado con:
  `docker compose -f poc\docker-compose.yml run --rm --no-deps --entrypoint /bin/sh n8n /home/node/.n8n-import/smoke.sh`.
- Resultado validado: 50 taskbots, Ola 1 = 7, Ola 2 = 29, Ola 3 = 14.
- Politica de revision validada: 27 gates de gobierno, 19 asistidos por IA/aprobacion dirigida,
  8 de evaluacion manual profunda.
- Salida n8n valida sensibilidad de umbrales y destinos objetivo post habilitacion API.
